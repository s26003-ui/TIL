# GitHub
---
## GitHub란? 
개발자들이 소스 코드를 인터넷에 저장하고 공유할 수 있는 플랫폼이다.코드의 변경 이력을 관리하는 **Git**을 기반으로 동작하며, 여러 사람이 함께 프로젝트를 진행할 때 협업을 쉽게 만들어 준다.
또한 전 세계 개발자들이 프로젝트를 공개하여 서로 참고하거나 기여할 수 있는 오픈소스 생태계가 활발하게 이루어지는 공간이기도 하다.
## GitHub의 주요 기능
1.버전관리: GitHub는 Git을 이용해 코드의 수정 기록을 저장한다.이 기능을 통해 코드가 언제 어떻게 변경되었는지 확인할 수 있으며, 문제가 발생했을 때 이전 상태로 되돌리는 것도 가능하다.

2.협업 기능: 여러 명의 개발자가 동시에 하나의 프로젝트를 작업할 때 GitHub를 사용하면 코드 충돌을 줄이고 작업 내용을 쉽게 공유할 수 있다.
또한 다른 사람이 작성한 코드를 확인하고 의견을 남기는 코드 리뷰도 가능하다.

3.오픈소스 공유: 많은 개발자들이 자신의 프로젝트를 공개하여 다른 사람들과 지식을 공유한다.
이러한 공개 프로젝트를 통해 다양한 기술과 개발 방식을 배울 수 있다.

4.개발자 활동 기록
GitHub는 개발자의 프로젝트와 활동 내역을 기록하기 때문에 개인 포트폴리오로도 활용된다.
기업에서는 지원자의 GitHub를 참고하여 개발 경험과 프로젝트 활동을 확인하기도 한다.

## Branch 사용하기

Branch는 기존 코드에 영향을 주지 않고 새로운 작업을 진행하기 위한 독립적인 작업 공간이다.
보통 기본 브랜치를 유지하면서 새로운 기능 개발이나 수정 작업을 별도의 브랜치에서 진행한다.
Branch는 기존 코드를 유지하면서 새로운 기능을 개발하기 위해 사용하는 독립적인 작업 공간이다.

git add . 변경된 파일을 스테이징 영역에 추가
git commit -m 변경 사항을 기록
git push origin (브랜치명)원격 저장소에 업로드

## Pull Request(PR)

Pull Request는 내가 작업한 브랜치를 다른 브랜치에 병합해 달라고 요청하는 기능이다.

#### 생성 방법

1.GitHub 저장소 페이지로 이동

2.Compare & pull request 버튼 클릭

3.병합할 브랜치(base)와 작업 브랜치(compare) 확인

4.제목과 설명 작성 후 Create pull request 선택

PR을 통해 다른 개발자들이 코드 내용을 확인하고 의견을 남길 수 있다.
## 코드 리뷰 과정

Pull Request가 생성되면 팀원들이 코드를 확인하고 피드백을 남길 수 있다.

리뷰 확인 방법

PR 페이지의 Files changed 탭에서 수정된 코드와 리뷰 내용을 확인할 수 있다.
## Issue 기능

Issue는 프로젝트에서 해야 할 작업이나 발생한 문제를 기록하고 관리하기 위한 기능이다.

#### 생성 방법

1.저장소 상단의 Issues 탭 클릭

2.New issue 선택

3.제목과 내용을 작성 후 Submit new issue 클릭

또한 담당자를 지정하거나 라벨을 설정하여 작업을 체계적으로 관리할 수 있다.
